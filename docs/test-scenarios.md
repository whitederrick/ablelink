# 테스트 시나리오

> 체크박스를 클릭하거나 `[x]`로 표시하여 진행 현황을 추적하세요.
> **테스트 환경**: https://able-link.co.kr (또는 로컬 http://localhost:3000)

---

## 테스트 계정 준비

| 구분 | 아이디 | 비밀번호 | 비고 |
|------|--------|----------|------|
| 슈퍼 관리자 | (직접 입력) | - | AdminRole=ADMIN |
| 에이전시 관리자 | (직접 입력) | - | AdminRole=AGENCY |
| 직무지도원 | (직접 입력) | - | 배정 현장 있는 계정 |

---

## 1. 관리자 인증

### 1-1. 로그인
- [ ] 올바른 아이디/비밀번호로 로그인 → 대시보드 진입 확인
- [ ] 잘못된 비밀번호로 로그인 → 오류 메시지 표시 확인
- [ ] 빈 칸으로 로그인 시도 → 폼 검증 오류 확인
- [ ] 로그인 10회 초과 시도 → Rate Limit 메시지 확인
- [ ] 로그아웃 → 세션 쿠키 삭제 + 로그인 페이지 리다이렉트 확인

### 1-2. 세션 유지
- [ ] 로그인 후 페이지 새로고침 → 세션 유지 확인
- [ ] 브라우저 탭 닫고 재접속 → 세션 유지 확인 (8시간 이내)
- [ ] `/admin` 직접 접근 시 비인증 → `/admin/login` 리다이렉트 확인

---

## 2. 관리자 대시보드

- [ ] 대시보드 로딩 확인 (오늘 출근 현황, 미확인 근태, 보고서 현황)
- [ ] 배정 종료 임박 목록 표시 확인 (D-10 이내)
- [ ] 미배정 현장 목록 표시 확인
- [ ] 운영 리스크 알림 카드 표시 확인
- [ ] 에이전시 관리자로 로그인 시 자기 에이전시 데이터만 표시 확인

## 3. 현장 관리

### 3-1. 현장 등록
- [ ] 현장명·주소 입력 → 주소 검색 API 동작 확인
- [ ] GPS 좌표 자동 입력 확인
- [ ] GPS 허용 반경 기본값 100m 확인
- [ ] 현장 등록 완료 → 목록에 반영 확인
- [ ] 필수 항목 누락 시 오류 표시 확인

### 3-2. 현장 조회/수정
- [ ] 현장 목록 조회 + 검색(이름) 확인
- [ ] 현장 상세 진입 → 담당 직무지도원, 훈련생 목록 표시 확인
- [ ] 현장 정보 수정 후 저장 확인
- [ ] 현장 비활성화 → 목록에서 제외 확인

### 3-3. GPS 기준점
- [ ] 기준점 등록 (관리자 확정) 확인
- [ ] 기준점 수정 후 반영 확인

## 12. 직무지도원 — 가입

### 12-1. 셀프 가입 (OTP)
- [ ] `/worker/signup` 접속 확인
- [ ] 전화번호 입력 → `인증번호 받기` 클릭 → 발송 확인
- [ ] 잘못된 전화번호 → 오류 메시지 확인
- [ ] 이미 가입된 전화번호 → 중복 오류 확인
- [ ] 인증번호 5분 만료 후 재발송 버튼 표시 확인
- [ ] 올바른 인증번호 입력 → 정보 입력 단계 이동 확인
- [ ] 틀린 인증번호 입력 → 오류 메시지 확인
- [ ] 이름(2자 이상)·비밀번호(8자 이상) 입력 → 약관 단계 이동 확인
- [ ] 필수 약관 미동의 시 완료 버튼 비활성 확인
- [ ] 전체 동의 버튼 동작 확인
- [ ] 약관 `보기` 링크 → `/terms`, `/privacy` 새 탭 열기 확인
- [ ] 가입 완료 → 자동 로그인 + `/worker/site/register` 이동 확인

### 12-2. 초대 링크 가입
- [ ] 관리자가 생성한 초대 링크 접속 → 에이전시명·전화번호 표시 확인
- [ ] 만료된 초대 링크 접속 → 오류 페이지 표시 확인
- [ ] 이미 사용된 초대 링크 → 오류 표시 확인
- [ ] 틀린 인증번호 입력 → 즉시 오류 확인 (서버 검증)
- [ ] 올바른 인증번호 → 정보 입력 이동 확인
- [ ] 이름 미리 채워짐 확인 (관리자가 입력한 경우)
- [ ] 가입 완료 → siteId 지정 시 `/worker/home`, 미지정 시 `/worker/site/register` 이동 확인

## 21. 보안 테스트

### 21-1. 인증 우회 시도
- [ ] 로그인 없이 `/admin` 직접 접근 → 로그인 리다이렉트 확인
- [ ] 로그인 없이 `/worker/home` 직접 접근 → 로그인 리다이렉트 확인
- [ ] 만료된 세션 쿠키로 API 요청 → 401 응답 확인
- [ ] 직무지도원 쿠키로 관리자 API 호출 → 401/403 확인

### 21-2. 에이전시 스코프 격리
- [ ] 에이전시 A 관리자가 에이전시 B의 직무지도원 ID로 API 호출 → 404/403 확인
- [ ] 에이전시 A 관리자 대시보드에 에이전시 B 데이터 미표시 확인

### 21-3. Rate Limiting
- [ ] 로그인 API 10회 이상 빠르게 시도 → 429 응답 + 재시도 시간 메시지 확인
- [ ] OTP 발송 API 연속 요청 → 429 응답 확인

### 21-4. 입력 검증
- [ ] 전화번호 필드에 문자 입력 → 오류 또는 무시 확인
- [ ] 비밀번호 7자 이하 → 가입 불가 확인
- [ ] 이름 1자 → 가입 불가 확인
- [ ] XSS 시도 (`<script>alert(1)</script>` 입력) → 이스케이프 처리 확인
